# CLAUDE.md — Stash to Project

AI assistant guide for the **Stash to Project** codebase.

---

## Project Overview

**Stash to Project** is a crochet management web app with AI assistance. Users log their yarn stash, get AI-generated project suggestions based on available materials, generate full step-by-step crochet patterns, and manage size profiles for themselves and others.

- **Language**: JavaScript (JSX) — no TypeScript
- **Framework**: Next.js 14 (App Router)
- **Database + Auth**: Supabase (PostgreSQL with Row Level Security)
- **AI**: Anthropic Claude API (`claude-sonnet-4-20250514`)
- **PWA**: Progressive Web App with service worker and manifest
- **UI language**: Spanish (all UI copy, comments, and prompts are in Spanish)

### Page architecture pattern

Each page route (`app/*/page.jsx`) is a thin server component that just renders its corresponding feature component:

```jsx
import FeatureComponent from "@/components/FeatureComponent";
export const metadata = { title: "..." };
export default function Page() { return <FeatureComponent />; }
```

All actual logic lives in the component files in `/components/`.

## Environment Setup

### Required environment variables

Copy `.env.local.example` to `.env.local` and fill in:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...
ANTHROPIC_API_KEY=sk-ant-api03-...
```

**Critical**: `ANTHROPIC_API_KEY` must never be prefixed with `NEXT_PUBLIC_` — it is server-only and used exclusively in API routes to keep it out of the client bundle.

### Database setup

Run `sql/001_initial_schema.sql` once in the Supabase Dashboard SQL Editor. This creates:

| Table | Purpose |
|---|---|
| `yarns` | User's yarn stash entries |
| `saved_projects` | AI-suggested projects the user saved |
| `size_profiles` | Body measurement profiles |
| `saved_patterns` | Full patterns generated by Claude |

All tables use Row Level Security (RLS) — every query is scoped to `auth.uid() = user_id`. There are no admin overrides; the Supabase anon key is sufficient.

### OAuth providers

Configure Google and GitHub OAuth in the Supabase Dashboard under Authentication → Providers. The redirect URI must be `https://<project>.supabase.co/auth/v1/callback`.

## Key Conventions

### Styling

- **No CSS framework** (no Tailwind, no CSS modules). All styles are inline JavaScript objects.
- **CSS custom properties** defined in `app/globals.css` — always use these variables:
  - `--ink`, `--paper`, `--cream`, `--rust`, `--rust-light`, `--rust-bg`
  - `--muted`, `--border`, `--white`, `--green`, `--nav-height`
- **Typography**: `'Cormorant Garamond'` (serif) for headings/brand, `'Outfit'` (sans-serif) for body
- **Brand accent color**: `var(--rust)` (`#B85C38`) for primary actions and active states
- Do not introduce CSS classes or external styling libraries unless explicitly requested

### Component patterns

- All feature components start with `"use client"` — they use `useState`, `useEffect`, Supabase auth
- Feature components are large single-file components (1000–2000 lines), not split into sub-files
- Sub-components are defined in the same file as the parent, below the main export
- Path alias `@/` resolves to the repo root (configured in `jsconfig.json`)

### Claude API integration

All Claude calls go through Next.js API routes — never call the Anthropic API directly from client-side code.

| Route | Behavior | Use case |
|---|---|---|
| `POST /api/analyze-stash` | Returns JSON | Stash project suggestions |
| `POST /api/generate-pattern` | Server-Sent Events stream | Web pattern generation |
| `POST /api/generate-pattern-full` | Returns JSON | React Native / non-streaming clients |

**Prompt management**: All prompts live in `lib/claudePrompts.js`. When modifying AI behavior, edit only the prompt builder functions there:
- `buildPatternPrompt({ garment, yarnWeight, yarnFiber, hookSize, skillLevel, style, colorCount, measurements, extraNotes })` — returns a detailed crochet pattern prompt in Spanish
- `buildStashAnalysisPrompt({ yarns, skillLevel })` — returns a prompt requesting exactly 4 project suggestions as valid JSON

**Model**: `claude-sonnet-4-20250514` with `anthropic-version: 2023-06-01`

### Supabase client

`lib/supabase.js` exports a single `supabase` client instance created with `createClient`. Import it as:

```js
import { supabase } from "@/lib/supabase";
```

This is the browser client. API routes do not use this client — they call Anthropic directly using `process.env.ANTHROPIC_API_KEY`.

### Database queries

Always filter by `user_id` matching the authenticated user. RLS enforces this at the DB level, but include it in queries for clarity. Example:

```js
const { data } = await supabase
  .from("yarns")
  .select("*")
  .eq("user_id", session.user.id);
```

### Authentication flow

1. Unauthenticated users land on `app/page.jsx` (the login page)
2. OAuth via `supabase.auth.signInWithOAuth({ provider: "google" | "github" })`
3. After auth, redirect to `/stash`
4. `Nav.jsx` listens to `supabase.auth.onAuthStateChange` to update UI reactively
5. Session is read with `supabase.auth.getSession()`

## Database Schema Reference

### `yarns`
| Column | Type | Notes |
|---|---|---|
| id | uuid | PK, auto-generated |
| user_id | uuid | FK → auth.users |
| name | text | Optional yarn name |
| weight | text | e.g. "Worsted", "DK" |
| fiber | text | e.g. "Algodón", "Lana" |
| color_name | text | Display color name |
| color_hex | text | e.g. "#C4704F" |
| meters | integer | Meters per skein |
| skeins | integer | Number of skeins |
| created_at | timestamptz | Auto |

### `saved_projects`
| Column | Type | Notes |
|---|---|---|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| emoji, name, description, difficulty, estimated_time, tip | text | Project fields |
| yarns_needed | jsonb | Array of yarn strings |
| stitches | jsonb | Array of stitch names |

### `size_profiles`
| Column | Type | Notes |
|---|---|---|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| name | text | Profile label |
| relation | text | e.g. "Para mí", "Mamá" |
| avatar_emoji | text | Default: "🧍" |
| chest, waist, hips, ... | numeric | All measurements in cm |
| size_standard, size_shoes, size_hat | text | Standard sizing fields |
| notes | text | Free-form notes |
| updated_at | timestamptz | Auto-updated via trigger |

### `saved_patterns`
| Column | Type | Notes |
|---|---|---|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| title, garment, skill_level, yarn_weight, hook_size, style | text | Pattern metadata |
| content | text | Full pattern in Markdown |

## What Not to Do

- Do not add TypeScript — the project is intentionally plain JavaScript
- Do not add CSS frameworks or external UI libraries
- Do not call the Anthropic API from client components — always use the API routes
- Do not commit `.env.local` — it is gitignored
- Do not expose `ANTHROPIC_API_KEY` in client-side code or prefix it with `NEXT_PUBLIC_`
- Do not run `scripts/generate-icons.js` during CI/build — it requires `sharp` which is not in devDependencies
- Do not add new SQL migrations without a numbered prefix (e.g. `002_...`) and without running them manually in Supabase
